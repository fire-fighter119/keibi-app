### ✅ 2. 📋 確認事項　（参照用）
> 現在のアプリにない → 新規追加。内容はNotionから。

連絡方法について
- 警備室へ電話切替後→環境局の方からの連絡は事務所へ伝える。警備室の電話でのみ対応可能。
- 電話・ピッチは「0」を押してからじゃないとかからない
- 関係者の連絡網は壁に貼ってある
- 関係機関の連絡先は引き出しの中（警察・消防は110・119じゃないので注意！）
決め事
在庫管理について
- 用紙が少なくなったら会社へLINEで連絡（他の人と被ってもOK）
- 打刻紙が残り1箱になったら報告
- のりなどの備品も残量確認
- 紙を持ってきたらパンチしてから収納

ごみについて
- ゴミは仕事の紙類のみ捨ててよい

入口ゲートの鍵の管理について
- 三井開発：鍵があっても勝手に入れる（施錠していてもOK）
- 従業員：営業時は鍵・ゲートを開けてあげる（巡回時に開錠）
- 他の業者：鍵のみを外しておいてあげる
- 他の業者＋三井開発が両方来る場合：鍵ごと警備室へ回収（三井開発が閉め忘れと勘違いして施錠してしまうため）ゲートは閉めたまま

## 開発引き継ぎメモ（Codex / Claude Code 共有用）

このアプリは、紙の警備チェック表をスマホで使いやすくするための単一HTMLアプリです。
今後、確認事項や現場ルールは増える想定です。修正時は下記を前提にしてください。

### 基本構成

- アプリ本体は `index.html`
- HTML / CSS / JavaScript は `index.html` 内で完結
- `design.md` は設計メモ・引き継ぎ用
- チェック状態は `localStorage` に保存
- PWA / オフライン利用を想定

### 重要な既存機能

- `勤務開始`ボタン
  - 次の勤務開始時に使う
  - 手順やメモは残す
  - チェック状態だけを一括で外す
- 編集モード
  - 項目の追加・削除ができる
  - 同じ小見出し内でドラッグ並び替えができる
- 巡回リスト
  - `施錠 / 維持 / 解錠` の3パターン
  - 毎回パターンを選んで開く
- LINE送信
  - 上番報告・下番報告の文章を送れる
  - iPhoneの仕様上、写真＋文章の完全自動送信は不可

### デザイン方針

- スマホでの現場利用を最優先
- 片手で押しやすく、見落としにくくする
- 重要作業は赤字・強調
- 装備品、例外、対応不要、要注意は色やラベルで区別
- 全完了時はゴールドの `MISSION COMPLETE` スタンプ表示
- 雰囲気はレトロRPGのクエスト手帳風
  - ただし特定作品のコピーはしない

### 修正ルール

- 既存機能を壊さない
- いきなり大改造しない
- 指示された範囲を中心に小さく直す
- 文言は現場で迷わないように推敲する
- 漢字が続きすぎる場合は、ひらがな・改行・チップ表示で読みやすくする
- チェックが必要なものと、単なる補足メモを混ぜない
- 重要なことを強調しすぎて、最重要事項が埋もれないようにする

### 今後増えそうな内容

- 確認事項
- 電話対応ルール
- 業者・三井開発・従業員ごとの対応ルール
- 巡回リストの順番やコメント
- 施錠 / 維持 / 解錠パターンの変更
- 帰る前チェック
- 引き継ぎ時の注意点

### 注意

`index.html` は保存済みユーザー状態を `localStorage` から復元します。
既存ユーザーに変更を反映するため、必要に応じて `patchKnownContent()` や同期処理も更新してください。
